# @repo/ui

Shared UI: [shadcn/ui](https://ui.shadcn.com/) primitives on
[Base UI](https://base-ui.com/), the Rams components, the theme provider, and the
Tailwind base stylesheet.

A leaf package — it depends on nothing else in the repo.

## Exports

```ts
import { Button } from "@repo/ui/components/button";
import { ThemeProvider } from "@repo/ui/lib/theme-provider";
import { cn } from "@repo/ui/lib/utils";
import "@repo/ui/styles/base.css";
```

| Path              | What's in it                                                                                         |
| ----------------- | ---------------------------------------------------------------------------------------------------- |
| `components/`     | Vendored shadcn/ui primitives, plus the Rams compositions (`rams-core`, `rams-content`, `rams-nav`). |
| `lib/`            | `theme-provider.tsx` (light/dark via `[data-theme]`), `utils.ts` (`cn`).                             |
| `hooks/`          | `use-mobile.ts`.                                                                                     |
| `styles/base.css` | Tailwind base and the design tokens.                                                                 |

## Adding a component

```sh
vpr ui add dialog        # into this package
vpr ui:web add dialog    # into apps/web instead
```

Both wrap the shadcn CLI; config is in `components.json`.

## Vendored primitives are lint-ignored

`packages/ui/components/` is excluded from Oxlint (see the comment in
[`vite.config.ts`](../../vite.config.ts)). Those files are generated by the
shadcn CLI and updated by re-running it, so local fixes are silently lost on the
next `add`. Keep your own compositions outside that directory, where they're
actually linted.

## Design language

The default theme is **Rams Document**: warm paper, near-black ink, one
restrained Braun-orange accent, hairline rules, square corners, Archivo
(grotesk) for text and Space Mono for labels. Light and dark both work; the theme
is applied via `[data-theme]` on `<html>`.

`apps/marketing` shares the language but not the code — it re-authors its
sections as `.astro` components rather than pulling React into a static site.
